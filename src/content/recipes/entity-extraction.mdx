## What This Does

Identifies and extracts structured entities — names, dates, locations, organizations, monetary amounts — from unstructured text. The output is structured JSON you can store directly in a database or pass to downstream systems.

### When to Use This

- Parsing resumes for candidate names, skills, and experience
- Extracting people, places, and dates from news articles
- Processing legal documents for parties, dates, and clauses
- Building knowledge graphs from unstructured content
- Enriching CRM records from emails

### How It Compares to Traditional Approaches

Traditional NER (spaCy, Stanford NLP) is fast and cheap but limited to pretrained entity types. LLMs can extract **any entity type you define** — just describe it in the prompt. They also handle context better ("Apple" the company vs. "apple" the fruit).

## Code

### Python

```python
import openai
import json

client = openai.OpenAI(api_key="YOUR_API_KEY")

ENTITY_SCHEMA = {
    "persons": ["string"],
    "organizations": ["string"],
    "locations": ["string"],
    "dates": ["string"],
    "amounts": ["string"],
}


def extract_entities(text):
    """
    Extract named entities from text.
    Returns a dict with entity types as keys and lists of strings as values.
    """
    try:
        response = client.chat.completions.create(
            model="gpt-4",
            messages=[
                {
                    "role": "system",
                    "content": (
                        "Extract all named entities from the text. "
                        "Return a JSON object with these keys: persons, organizations, "
                        "locations, dates, amounts. Each key maps to an array of strings. "
                        "If no entities found for a key, use an empty array. "
                        "Return only valid JSON, no explanation."
                    ),
                },
                {"role": "user", "content": text},
            ],
            temperature=0,
            max_tokens=1000,
        )
        return json.loads(response.choices[0].message.content)
    except json.JSONDecodeError:
        print("Warning: model returned invalid JSON")
        return None
    except openai.APIError as e:
        print(f"API error: {e}")
        return None


# --- Example usage ---

text = """
On January 15, 2024, Acme Corp announced that CEO Jane Smith would open a
new research facility in Berlin, Germany. The project has a budget of €2.5 million
and is backed by the European Innovation Council. Dr. Michael Chen from Stanford
University will lead the AI research division starting March 1st.
"""

entities = extract_entities(text)
print(json.dumps(entities, indent=2))

# Expected output:
# {
#   "persons": ["Jane Smith", "Michael Chen"],
#   "organizations": ["Acme Corp", "European Innovation Council", "Stanford University"],
#   "locations": ["Berlin", "Germany"],
#   "dates": ["January 15, 2024", "March 1st"],
#   "amounts": ["€2.5 million"]
# }
```

### Node.js

```javascript
import OpenAI from "openai";

const client = new OpenAI({ apiKey: "YOUR_API_KEY" });

/**
 * Extract named entities from text.
 * @param {string} text - The input text.
 * @returns {Promise<Object|null>} Entities grouped by type, or null on error.
 */
async function extractEntities(text) {
  try {
    const response = await client.chat.completions.create({
      model: "gpt-4",
      messages: [
        {
          role: "system",
          content:
            "Extract all named entities from the text. Return a JSON object with keys: persons, organizations, locations, dates, amounts. Each key maps to an array of strings. If none found for a key, use an empty array. Return only valid JSON.",
        },
        { role: "user", content: text },
      ],
      temperature: 0,
      max_tokens: 1000,
    });

    return JSON.parse(response.choices[0].message.content);
  } catch (error) {
    if (error instanceof SyntaxError) {
      console.error("Invalid JSON from model");
    } else {
      console.error("API error:", error.message);
    }
    return null;
  }
}

// --- Example usage ---

const text = `On January 15, 2024, Acme Corp announced that CEO Jane Smith
would open a new research facility in Berlin, Germany.`;

const entities = await extractEntities(text);
console.log(JSON.stringify(entities, null, 2));
```

## How It Works

1. **System prompt defines the exact JSON schema**: the model knows what keys to return and what type each value should be
2. **"Return only valid JSON"**: prevents the model from wrapping the output in explanation text
3. **Temperature 0**: ensures deterministic, consistent extraction
4. **Empty arrays for missing keys**: keeps the schema consistent even when some entity types aren't present

### Key Parameters

| Parameter     | Value | Why                                                    |
|---------------|-------|--------------------------------------------------------|
| `temperature` | 0     | Extraction needs precision, not creativity             |
| `max_tokens`  | 1000  | Long texts can produce many entities                   |
| `model`       | gpt-4 | Best at understanding context and avoiding false matches |

## Adding Custom Entity Types

Just modify the system prompt to include your custom types:

```
"Extract: persons, organizations, locations, dates, amounts, product_names, skill_sets.
Each key maps to an array of strings."
```

The model will adapt without any retraining.

## Tips

- **Always wrap JSON.parse in try/catch**: the model occasionally returns malformed JSON, especially with complex nested entities
- **Use JSON mode** when available: `response_format: { type: "json_object" }` guarantees valid JSON output on GPT-4
- **For long documents**: process paragraph by paragraph and merge results — deduplicating entities across chunks
- **Disambiguation**: if "Apple" could be a company or a fruit, add context to the prompt: "Focus on business and financial entities"
- **Validation**: check that returned entity types match your schema — discard unexpected keys

## Common Pitfalls

| Problem | Cause | Fix |
|---------|-------|-----|
| Invalid JSON response | Model adds explanation text | Add "Return only valid JSON, no explanation" |
| Missed entities | Text too long, model skips later parts | Process in chunks |
| Duplicate entities | Same entity mentioned multiple ways | Deduplicate in post-processing |
| Wrong entity type | Ambiguous context | Add domain context to prompt |
| Partial dates | "next Tuesday" instead of full date | Add "Normalize dates to YYYY-MM-DD format" |
