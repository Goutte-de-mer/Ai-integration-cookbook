## What This Does

Converts messy, unstructured text — emails, invoices, reports, web pages — into clean, typed JSON that your application can process directly. Define the shape of the data you want, and the LLM fills it in.

### When to Use This

- Parsing invoices and receipts into structured line items
- Extracting contact info from email signatures
- Converting job postings into structured fields (title, salary, requirements)
- Processing form responses or survey free-text into categories
- Building data pipelines from unstructured sources (PDFs, emails, chat logs)

### How It Compares to Traditional Approaches

Regex and rule-based parsers work for highly consistent formats but break on variations. LLMs handle **messy, inconsistent formats** and adapt to new layouts without rewriting rules.

### Python

```python
import openai
import json

client = openai.OpenAI(api_key="YOUR_API_KEY")

INVOICE_SCHEMA = """{
  "vendor": "string",
  "invoice_number": "string",
  "date": "YYYY-MM-DD",
  "items": [
    {
      "description": "string",
      "quantity": number,
      "unit_price": number,
      "total": number
    }
  ],
  "subtotal": number,
  "tax": number,
  "total": number,
  "currency": "string (ISO 4217)"
}"""


def extract_structured_data(text, schema=INVOICE_SCHEMA, context="an invoice"):
    """
    Extract structured data from unstructured text.

    Args:
        text: The unstructured input text.
        schema: JSON schema string describing the desired output.
        context: Brief description of what the text is (e.g., "an invoice").
    Returns:
        Parsed JSON dict, or None on error.
    """
    try:
        response = client.chat.completions.create(
            model="gpt-4",
            messages=[
                {
                    "role": "system",
                    "content": (
                        f"Extract data from {context} into the following JSON structure:\n"
                        f"{schema}\n\n"
                        "Rules:\n"
                        "- Return only valid JSON, no markdown or explanation\n"
                        "- Use null for fields that cannot be determined\n"
                        "- Normalize dates to YYYY-MM-DD\n"
                        "- Use numbers (not strings) for numeric fields\n"
                        "- Currency should be ISO 4217 code (USD, EUR, GBP)"
                    ),
                },
                {"role": "user", "content": text},
            ],
            temperature=0,
            max_tokens=2000,
        )

        raw = response.choices[0].message.content
        # Strip markdown fences if model adds them
        if raw.startswith("```"):
            raw = "\n".join(raw.split("\n")[1:-1])
        return json.loads(raw)

    except json.JSONDecodeError as e:
        print(f"Invalid JSON: {e}")
        return None
    except openai.APIError as e:
        print(f"API error: {e}")
        return None


# --- Example usage ---

invoice_text = """

### Node.js

```javascript
import OpenAI from "openai";

const client = new OpenAI({ apiKey: "YOUR_API_KEY" });

const INVOICE_SCHEMA = `{
  "vendor": "string",
  "invoice_number": "string",
  "date": "YYYY-MM-DD",
  "items": [{ "description": "string", "quantity": number, "unit_price": number, "total": number }],
  "subtotal": number,
  "tax": number,
  "total": number,
  "currency": "string (ISO 4217)"
}`;

/**
 * Extract structured data from unstructured text.
 * @param {string} text - The unstructured input.
 * @param {string} schema - JSON schema describing desired output.
 * @param {string} context - Description of the text type.
 * @returns {Promise<Object|null>} Parsed structured data, or null on error.
 */
async function extractStructuredData(text, schema = INVOICE_SCHEMA, context = "an invoice") {
  try {
    const response = await client.chat.completions.create({
      model: "gpt-4",
      messages: [
        {
          role: "system",
          content: `Extract data from ${context} into this JSON structure:\n${schema}\n\nReturn only valid JSON. Use null for unknown fields. Normalize dates to YYYY-MM-DD.`,
        },
        { role: "user", content: text },
      ],
      temperature: 0,
      max_tokens: 2000,
    });

    let raw = response.choices[0].message.content;
    if (raw.startsWith("```")) {
      raw = raw.split("\n").slice(1, -1).join("\n");
    }
    return JSON.parse(raw);
  } catch (error) {
    console.error("Error:", error.message);
    return null;
  }
}

// --- Example usage ---

const data = await extractStructuredData(
  "Invoice #123 from Acme Corp, dated Jan 5 2024. 2x Widget at $50 each. Total: $100 + $8 tax = $108."
);
console.log(JSON.stringify(data, null, 2));
```

## How It Works

1. **Schema in the prompt**: the model sees the exact JSON shape you want, including field types and format rules
2. **Explicit rules** (use null, normalize dates, use numbers) prevent the most common formatting issues
3. **Temperature 0**: ensures the model sticks strictly to what's in the text
4. **Post-processing**: strip markdown fences before parsing — models sometimes add them despite being told not to

### Key Parameters

| Parameter     | Value | Why                                                    |
|---------------|-------|--------------------------------------------------------|
| `temperature` | 0     | Extraction must be precise and deterministic           |
| `max_tokens`  | 2000  | Complex documents produce large JSON objects           |
| `model`       | gpt-4 | Best at following complex schemas accurately           |

## Reusable Schema Library

Define schemas once, reuse across your app:

```python
SCHEMAS = {
    "invoice": INVOICE_SCHEMA,
    "contact": '{"name": "string", "email": "string", "phone": "string", "company": "string"}',
    "job_posting": '{"title": "string", "company": "string", "salary_range": "string", "location": "string", "requirements": ["string"]}',
    "meeting_notes": '{"date": "YYYY-MM-DD", "attendees": ["string"], "agenda": ["string"], "action_items": [{"task": "string", "owner": "string", "due": "string"}]}',
}

# Use any schema:
data = extract_structured_data(text, schema=SCHEMAS["contact"], context="an email signature")
```

## Tips

- **JSON mode**: on GPT-4, use `response_format: { type: "json_object" }` to guarantee valid JSON output (no need for manual fence stripping)
- **Validation**: always validate the parsed output against your expected schema — check for required fields, correct types, reasonable values
- **Math verification**: for invoices, verify that `quantity × unit_price = total` and items sum to subtotal — LLMs sometimes copy numbers wrong
- **Fallback values**: use `null` rather than empty strings for missing fields — it's cleaner to check `if field is not None`
- **Batch processing**: for many similar documents, process in batches of 5–10 per API call to reduce overhead

## Common Pitfalls

| Problem | Cause | Fix |
|---------|-------|-----|
| Invalid JSON | Model adds explanation text | Use JSON mode, or strip markdown fences |
| Numbers as strings | Schema unclear | Specify "use numbers (not strings) for numeric fields" |
| Wrong date format | Ambiguous dates like "03/04/2024" | Specify "Normalize to YYYY-MM-DD" and note locale |
| Missing items in lists | Complex or long document | Process sections individually |
| Math doesn't add up | Model copies numbers incorrectly | Validate arithmetic in post-processing |
