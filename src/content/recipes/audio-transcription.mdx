## What This Does

Send an audio file to a speech-to-text model and get back a verbatim transcript of the spoken content. This enables meeting notes, podcast indexing, voice command processing, and accessibility features like captions.

### When to Use This

- **Meeting notes**: auto-transcribing recorded calls or standups.
- **Accessibility**: generating captions or subtitles for spoken content.
- **Voice input**: capturing dictated content for further processing by an LLM.

  python={`import openai

client = openai.OpenAI(api_key="YOUR_API_KEY")

def transcribe_audio(file_path):
    try:
        with open(file_path, "rb") as audio_file:
            response = client.audio.transcriptions.create(

            )

const client = new OpenAI({ apiKey: "YOUR_API_KEY" });

async function transcribe(filePath) {
  try {
    const response = await client.audio.transcriptions.create({
      model: "whisper-1",
      file: createReadStream(filePath),
    });
    return response.text;
  } catch (error) {
    return null;
  }
}

## How It Works

1. **Upload**: the audio file is sent as multipart form data.
2. **Acoustic model**: Whisper processes the waveform into tokens.
3. **Optional context**: use the \`prompt\` parameter to improve accuracy on technical jargon.

### Key Parameters

| Parameter | Value | Why |
| :--- | :--- | :--- |
| \`model\` | whisper-1 | High-accuracy speech-to-text model. |
| \`language\` | "en" / "fr" | Optional, but speeds up processing. |
| \`prompt\` | string | Context hint for names or acronyms. |

## Tips

- **File size**: limit is 25 MB per file. Split larger files with \`ffmpeg\`.
- **Post-process**: Whisper doesn't always add perfect punctuation; pipe the result into GPT-4 to clean it up.
